热火朝天~火冒三丈~的装修记录——在我的Gridea博客加入Waline评论系统

2022-08-23 Views 出埃及记2111字8 min read繁

宝宝都能懂，但你最sb的同事则不一定

2022.08.24更新：设置不显示评论者UA信息

前略因为换域名一时半会不太能有进展，于是我转而开始进行评论区的实装。最开始选择的是Gridea官方文档中推荐的Valine，总之比较顺利地让评论区显示出来了，一切看起来都很好，结果在测试时又出现无法解决的问题，又进行一番检索得知了Valine的一些缺点，于是最终还是决定使用Waline。

背景碎碎念

决定详细记录这个过程主要是因为，我发现我找到的所有资料都不是为我这种完全的门外汉准备的。
在这个过程中，我也看到不少人提出和我同样的问题，却没有得到充分的解答（最后回复时间：两年前）。当我找不到资料的时候，我一直在想为什么没有人，哪怕有一个人可以写得再详细一点呢？因此虽然知道这种攻略……呃……记录……很块就会过期，但我还是希望写下来。


WARN: 作者是幼稚园，这篇文章面向的也是没有任何码基础的宝宝，只用于记录“我是如何成功实装评论区的”，对各概念并不（能）进行准确的系统性解释，如果你有相关基础可能会觉得太过弱智。

本记录基于的环境是Gridea+github pages，主题为bitcron-pro。

一个能派上用场的小技巧


在网页中按F12查看网页源代码，点击最左上角的箭头，再点击网页上的任意部分就可以查看该部分对应的代码，怎么样，是不是很方便呢？来，试试看👉

PS. 我是在【Missing 不想睡】老师的Hugo | 装修小技巧里面学到的！


开始：部署

LeanCloud的部分和Vercel的部分，在官方文档中都确实讲得很详细，不再特意重复……以下是我在途中遇到的坑和需要注意的部分：

  • 务必使用国际版LeanCloud，因为国内版LeanCloud不仅需要手机号注册，还要求你的域名已通过ICP备案，真是离大谱。
  • 注册Vercel时直接用github账号注册，如果使用gmail注册，会被要求短信认证（可以使用中国大陆手机号）。但我注册的时候似乎跟github的连接出了问题，仍然出现了短信认证😅如果出现这种情况可以重新尝试。
  • 如果跟github的连接不稳定，在Deploy阶段可能会出问题，导致不能借助github的源码在vercel初始化仓库，最终会得到一个空仓库（如下图所示），无法进行剩下的步骤。从头试一次就好。多余的仓库可以点进仓库详情后拉到settings页面最下方删掉。
  • 部署完成点击visit时会弹出网页，内容就是一个评论栏，请保存这个网址，在引入步骤会作为serverURL使用。
  • 在上面的网址后面加上/ui/register 就进入了管理评论后台，第一个注册的人（你）默认成为管理员。

难关：引入

官方文档中提供了两种引入Waline客户端的方式，分别是通过CDN和通过NPM，好，一个都看不懂👍
用人话说，通过CDN=通过链接引用放在服务器上的源码，通过NPM=把源码下到本地使用。
这里选择的是通过CDN的方法。

又是一通过程碎碎念

最开始我以为把源码下到本地才能自定义样式，毕竟放在服务器上的源码我又修改不了，怎么搞嘛？于是逆天而行开始走npm。nodejs安装时会自带npm，于是从下载nodejs开始，不停输入网上看来的命令把nodejs和npm的版本都更新到最新，报错信息看不懂就不停丢到DeepL翻译，下下来之后还不知道文件去哪了（参见：npm 安装包的路径在哪里）结果下好之后，仍然不知道该怎么引用，被自己气晕

首先

打开主题文件夹bitcron-pro/templates 下的post.ejs
在开头的<head>和</head>之间，</head>上面插入

<script src="https://unpkg.com/@waline/client@v2.0.0/dist/waline.js"></script>
<link
  rel="stylesheet"
  href="https://unpkg.com/@waline/client@v2.0.0/dist/waline.css"
/>

v2.0.0这个字段是用来指定版本号的。如果不指定的话，会默认为最新版本。

然后

在接下来的<body>和</body>之间，<div class="doc_comments">的下面，可以看到原本配置在这里的gitalk和disqus的代码。

将它们替换成

  <div id="waline"></div>
  <script>
    Waline.init({
      el: '#waline',
      serverURL: 'https://your-domain.vercel.app（这个地方替换成部署环节第四条提到的网址）',
    });
  </script>

过程中可以依靠vscode的报错检查自己有没有复制漏/多什么东西，
此时保存后预览，应该就能看到博文页面上出现评论栏了。
只不过，它的配色主题是默认的绿色。


收尾：自定义样式

柳暗花明又一村！从CDN引入原来也是可以自定义样式的。

自定义配色

在上一部分的最后一段代码后面加入：

<style>
/*设置 Waline 的主题色*/
:root{
    --waline-theme-color:#74b2c2;
	--waline-active-color: #88c4d3;
	}
	</style>

waline-theme-color是整个评论栏的主题色，waline-active-color（据我观察）是鼠标悬停在发送按钮上时按钮的颜色。官方文档中还提供了很多变量，可以慢慢研究。

自定义提示信息

在<script>和Waline.init({之间加入：

    /*设置自定义文本*/
    const locale = {
        placeholder: '欢迎评论（未输入评论时的占位符）',
        sofa: '沙发（无评论时显示的字样）',
	};
        （Waline.init……serverURL 省略几行）
        locale,

官方文档中还提供了很多变量。

自定义表情

在serverURL:这一行下面加入

        // 设置 emoji
        emoji: [
        '//unpkg.com/@waline/emojis@1.0.1/weibo',
        '//unpkg.com/@waline/emojis@1.0.1/bilibili',
        ],

emoji下面的几行链接即为表情的链接。可以通过检索Waline emoji来寻找其他人制作的表情包。我加入了象常用的blob（来源见文末。

其他环境变量

我在测试的时候居然没有注意到Waline会默认显示评论者的UA信息😱今晚回到家紧急进行了一个加工！
服务端设置中提供了一些可以修改的环境变量，需要在Vercel的Settings - Environment Variables中设置。
以不显示评论者UA为例：
在NAME中输入DISABLE_USERAGENT，VALUE设置为TRUE，如图所示。
我设置了不显示评论者UA和归属地。向之前评论的朋友狠狠三跪九叩了……
设置完之后别忘了重新部署。进入Deployments选项卡进行再部署。我因为总共部署了三次，所以这里有三个，总之选择最新的那个进行重新部署就好。
重新部署之后，serverURL会改变，记得到之前用到这条地址的地方，即post.ejs里更新它，并用新地址进入后台系统，无需重新注册，之前的评论也会保留下来。
